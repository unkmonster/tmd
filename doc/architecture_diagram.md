## 图例说明

| 元素 | 含义 |
|------|------|
| `main.go` → `service` | 主调用链（自上而下） |
| `main.go` 分叉为 CLI / Server | 两种运行模式 |
| 所有路径汇聚到 `service` | 统一应用服务层，两种模式共享 |
| `service` → 三路分发 | 业务编排 / Twitter API / 数据库 |
| 底部基础设施层 | 下载、实体、命名、路径、工具 |
| 右侧支撑层 | 配置、日志、调度，不直接参与下载流程 |
